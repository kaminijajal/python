1) What is an API (Application Programming Interface)?
->An API (Application Programming Interface) is essentially a set of rules and tools that allows different software applications to communicate with each other. It acts as a bridge between different programs, letting them exchange data and use each other’s functionality without needing to know how the other program is implemented.

Here’s a breakdown:

1=>Interface – Think of it like a menu at a restaurant. The menu tells you what you can order, but you don’t need to know how the chef prepares the food. Similarly, an API shows what operations you can perform, without exposing the internal code.

2=>Requests and Responses – Most APIs work by sending a request to another system and receiving a response. For example, a weather app can request the current weather from a weather service API, and the API responds with the data.

3=>**Types of APIs

2)Types of APIs: REST, SOAP.
->REST APIUses HTTP requests (GET, POST, PUT, DELETE).
Data usually in JSON format.
Lightweight and easy to use.

Example: GET /users/1

->SOAP API
Uses XML for messages.
More strict and formal (has a standard protocol).
Built-in error handling and security.

Example: <soap:Envelope>...</soap:Envelope>
->In short: REST = simple & fast, SOAP = strict & feature-rich.

3)Why are APIs important in web development?
->APIs are important in web development because they allow different applications, services, or systems to communicate and share data easily. They make web apps more flexible, let developers reuse functionality instead of building everything from scratch, and enable features like login with Google, payment processing, or fetching data from a server.

In short: APIs make development faster, more efficient, and connected.

6)What is Serialization?
->Serialization is the process of converting complex data (like objects, Python dictionaries, or querysets) into a format that can be easily stored or transmitted, such as JSON or XML.

In simple words:

->Turning Python objects into a format that can be sent over the internet or saved to a file.
->For example, in Django REST Framework:

from rest_framework import serializers

class ProductSerializer(serializers.Serializer):
    name = serializers.CharField()
    price = serializers.FloatField()

->Here, ProductSerializer converts a Product object into JSON so it can be sent to a client.
->Serialization is basically “Python objects → JSON/XML for sharing.”

7)Converting Django QuerySets to JSON.

->1. Using values() + json.dumps:
from django.http import JsonResponse
from .models import Product

def products_json(request):
    qs = Product.objects.all().values()  # QuerySet as dict
    return JsonResponse(list(qs), safe=False)  # Convert to list for JSON

->2. Using Django serialize:
from django.core.serializers import serialize
from django.http import HttpResponse
from .models import Product

def products_json(request):
    qs = Product.objects.all()
    data = serialize('json', qs)
    return HttpResponse(data, content_type='application/json')

->JsonResponse is simpler for API responses.
->serialize keeps model metadata (like pk, model name).

8)Using serializers in Django REST Framework (DRF)
->In Django REST Framework (DRF), serializers are used to convert complex data types like Django models into JSON (for API responses) and also to validate and convert incoming JSON into Python objects (for creating/updating models).

->example
from rest_framework import serializers
from .models import Product

class ProductSerializer(serializers.ModelSerializer):

11)Understanding views in DRF: Function-based views vs Class-based views.
->1. Function-Based Views (FBV)

-You write a Python function to handle requests.
-Each HTTP method (GET, POST, PUT, DELETE) is handled inside the function.
-Simple and easy for small APIs.

from rest_framework.decorators import api_view
from rest_framework.response import Response

@api_view(['GET', 'POST'])
def product_list(request):
    if request.method == 'GET':
        return Response({"message": "List of products"})
    elif request.method == 'POST':
        return Response({"message": "Product created"})

2. Class-Based Views (CBV)

-You write a Python class that inherits from DRF classes.
-Each HTTP method is a separate method (get(), post(), etc.).
-Supports mixins and generic views, so less code for common operations (CRUD).

from rest_framework.views import APIView
from rest_framework.response import Response

class ProductList(APIView):
    def get(self, request):
        return Response({"message": "List of products"})

    def post(self, request):
        return Response({"message": "Product created"})

13)Adding pagination to APIs to handle large data sets
->1.Set pagination in settings.py:
REST_FRAMEWORK = {
    'DEFAULT_PAGINATION_CLASS': 'rest_framework.pagination.PageNumberPagination',
    'PAGE_SIZE': 10  # Number of items per page
}

->2. API view automatically supports pagination:

from rest_framework.views import APIView
from rest_framework.response import Response
from .models import Product
from .serializers import ProductSerializer
from rest_framework.pagination import PageNumberPagination

class ProductList(APIView):
    def get(self, request):
        products = Product.objects.all()
        paginator = PageNumberPagination()
        paginator.page_size = 10
        result = paginator.paginate_queryset(products, request)
        serializer = ProductSerializer(result, many=True)
        return paginator.get_paginated_response(serializer.data)

How it works:

-Request first page: /api/products/?page=1
-Request second page: /api/products/?page=2

Step 3: Send OTP
from twilio.rest import Client
import random

def send_otp(to_number):
    otp = random.randint(100000, 999999)  # Generate 6-digit OTP
    account_sid = 'YOUR_TWILIO_ACCOUNT_SID'
    auth_token = 'YOUR_TWILIO_AUTH_TOKEN'
    client = Client(account_sid, auth_token)

    message = client.messages.create(
        body=f"Your OTP is {otp}",
        from_='+1234567890',  # Twilio number
        to=to_number
    )
    print(f"OTP sent: {otp}")
    print(message.sid)
    return otp

# Example usage

from rest_framework.decorators import api_view
from rest_framework.response import Response
from rest_framework import status
from .models import Doctor
from .serializers import DoctorSerializer

# List all doctors / Add new doctor
@api_view(['GET', 'POST'])
def doctor_list(request):
    if request.method == 'GET':
        doctors = Doctor.objects.all()
        serializer = DoctorSerializer(doctors, many=True)
        return Response(serializer.data)
    elif request.method == 'POST':
        serializer = DoctorSerializer(data=request.data)
        if serializer.is_valid():
            serializer.save()
            return Response(serializer.data, status=status.HTTP_201_CREATED)
        return Response(serializer.errors, status=status.HTTP_400_BAD_REQUEST)

# Get, Update, Delete doctor by ID
@api_view(['GET', 'PUT', 'DELETE'])
def doctor_detail(request, pk):
    try:
        doctor = Doctor.objects.get(pk=pk)
    except Doctor.DoesNotExist:
        return Response({"error": "Doctor not found"}, status=status.HTTP_404_NOT_FOUND)

    if request.method == 'GET':
        serializer = DoctorSerializer(doctor)
        return Response(serializer.data)

    elif request.method == 'PUT':
        serializer = DoctorSerializer(doctor, data=request.data)
        if serializer.is_valid():
            serializer.save()
            return Response(serializer.data)
        return Response(serializer.errors, status=status.HTTP_400_BAD_REQUEST)

    elif request.method == 'DELETE':
        doctor.delete()
        return Response(status=status.HTTP_204_NO_CONTENT)

20)Difference between authentication and authorization.
->Authentication = "Who are you?" 
    It checks identity.
    Example: Logging in with username and password.

->Authorization = "What can you do?" 
    It checks permissions.
    Example: Admin can delete users, regular user cannot.

->In short: Authentication = verify identity, Authorization = verify access.

->Calculate Distance Between Two Locations (JavaScript)
<script>
function calculateDistance() {
  const service = new google.maps.DistanceMatrixService();

  service.getDistanceMatrix(
    {
      origins: ["Delhi, India"],
      destinations: ["Mumbai, India"],
      travelMode: google.maps.TravelMode.DRIVING,
      unitSystem: google.maps.UnitSystem.METRIC,
    },
    (response, status) => {
      if (status === "OK") {
        const distance =
          response.rows[0].elements[0].distance.text;
        const duration =
          response.rows[0].elements[0].duration.text;

        console.log("Distance:", distance);
        console.log("Duration:", duration);
      }
    }
  );
}
</script>
